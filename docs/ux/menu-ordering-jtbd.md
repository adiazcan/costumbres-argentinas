# Jobs to Be Done

## Primary Job Statement
When I am hungry and deciding what to eat from Costumbres Argentinas, I want to read the menu quickly on my phone or desktop, so I can either order with confidence by phone or show my choice to the waiter without confusion.

## Secondary Job Statements
- When I am already in the restaurant, I want the menu to be easy to scan from my phone, so I can decide fast and keep the table moving.
- When I am ordering remotely, I want the phone number, promos, and product details to be obvious, so I can call with my order already defined.

## Users
- On-site customer using a phone at the table
- Remote customer ordering from home or work on phone or desktop
- Occasional desktop user comparing options before calling

## Current Pain Points Observed
- The brand story appeared before the task-critical menu and contact flow.
- Phone ordering existed but was not the primary call to action.
- Promotions were parsed in code but not visible in the UI.
- The site did not clearly separate the in-store and remote ordering modes.
- Search feedback was weak, which slowed menu scanning.

## UX Direction Applied
- Promote task-first navigation: menu, promos, orders, history.
- Make phone ordering a first-class action in header, hero, contact, and mobile CTA.
- Add an explicit service-mode toggle for on-site vs remote use.
- Move the story section after menu and ordering actions.
- Add visible search/result feedback and quick summary context.
